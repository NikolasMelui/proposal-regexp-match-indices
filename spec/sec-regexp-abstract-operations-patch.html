RegExp Abstract Operations

MatchOptions Records

A MatchOptions is a Record value used to encapsulate options for a regular expression matching operation.

MatchOptions Records have the fields listed in .
Field Name Value Meaning
[[Capture]] `"strings"` or `"indices"` Whether the operation should capture substrings or the start/end indices of the capture.

FromMatchOptions ( _Options_ )

When the abstract operation FromMatchOptions is called with argument _Options_, the following steps are taken:

1. Assert: _Options_ is a MatchOptions Record. 1. Let _obj_ be ObjectCreate(%ObjectPrototype%). 1. Assert: _obj_ is an extensible ordinary object with no own properties. 1. Perform CreateDataProperty(_obj_, `"capture"`, _Options_.[[Capture]]). 1. Assert: All of the above CreateDataProperty operations return *true*. 1. Return _obj_.

ToMatchOptions ( _Options_ )

When the abstract operation ToMatchOptions is called with argument _Options_, the following steps are taken:

1. If _Options_ is *undefined* or *null*, then 1. Let _optionsObj_ be *undefined*. 1. Else, 1. Let _optionsObj_ be ? ToObject(_Options_). 1. Let _capture_ be ? GetOption(_optionsObj_, `"capture"`, String, `"strings"`, « `"strings"`, `"indices"` »). 1. Return a new MatchOptions { [[Capture]]: _capture_ }.
